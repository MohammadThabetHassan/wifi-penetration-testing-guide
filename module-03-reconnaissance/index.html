Wireless Reconnaissance with airodump-ng

Discover and catalog wireless networks and clients in range. Learn to interpret airodump-ng output, capture handshake files, and identify targets for subsequent attack modules.

Beginner
~40 min
Adapter required

Legal Notice: Reconnaissance must only be performed on networks you own or have explicit written authorization to test. Passive monitoring of wireless frequencies may be restricted in your jurisdiction.

02 Basic airodump-ng Commands

Start with a general scan, then lock to your target for focused reconnaissance.

General scan (channel hopping)
# General scan - hops through all channels kali@kali:~$ sudo airodump-ng wlan0mon
Targeted capture - lock to channel + write files
# Target specific AP and lock to its channel kali@kali:~$ sudo airodump-ng --bssid AA:BB:CC:DD:EE:FF -c 6 -w capture wlan0mon # --bssid = target AP MAC # -c = channel number # -w = write to files (capture-01.cap, capture-01.csv, etc.)
Scan specific band
# 2.4 GHz only kali@kali:~$ sudo airodump-ng --band g wlan0mon # 5 GHz only kali@kali:~$ sudo airodump-ng --band a wlan0mon # Both bands kali@kali:~$ sudo airodump-ng --band abg wlan0mon

04 Client (Station) Enumeration

Clients reveal critical intelligence — the networks they've connected to, their MAC addresses, and traffic patterns.

🔍
Key insight: Even when a network hides its SSID (doesn't broadcast), clients that have connected before probe for it by name — revealing the hidden SSID in the process.
Client Station Output
BSSID STATION PWR Rate Probes
AA:BB:CC:DD:EE:FF 11:22:33:44:55:66 -50 54e-1e
AA:BB:CC:DD:EE:FF 22:33:44:55:66:77 -60 54e-1e
(not associated) 33:44:55:66:77:88 -55 54e-1e HomeWiFi, OfficeNet, Starbucks
⚡
Practical use: The "Probes" column reveals networks a client has previously connected to. Use these for Evil Twin attacks — create an AP with the same name and perform a deauth to force the client to connect to you.

05 Hidden SSID Detection

Networks with hidden SSIDs still appear in airodump-ng, but the ESSID column is blank.

🔎 Hidden Network in airodump-ng

BSSID PWR Beacons CH ENC ESSID
AA:BB:CC:DD:EE:FF -45 120 6 WPA2 (blank)

Revealing the Hidden SSID

Even when the ESSID is hidden in beacons, it appears in cleartext during the authentication/association process:

Wait for client connection or force one
# Monitor the hidden AP - wait for client to reconnect kali@kali:~$ sudo airodump-ng --bssid AA:BB:CC:DD:EE:FF -c 6 wlan0mon # Watch ESSID column - it will populate when client connects # Or force a deauth to trigger re-authentication (Module 08) kali@kali:~$ sudo aireplay-ng --deauth 10 -a AA:BB:CC:DD:EE:FF wlan0mon
⚠
Limitation: If a hidden network has zero active clients and no new clients connect during your capture session, you cannot determine the SSID.

06 Writing Capture Files

Capture files are essential for offline analysis, handshake storage, and evidence gathering.

Writing capture files
# Basic capture - creates capture-01.cap, capture-01.csv, etc. kali@kali:~$ sudo airodump-ng -w capture wlan0mon # PCAP format (recommended for Wireshark) kali@kali:~$ sudo airodump-ng --output-format pcap -w capture wlan0mon # Kismet format (for Kismet integration) kali@kali:~$ sudo airodump-ng --output-format kismet -w capture wlan0mon

Checking Capture Contents

Verify capture file
# View packets with tcpdump kali@kali:~$ tcpdump -r capture-01.cap | head -20 # Check for handshake with aircrack-ng kali@kali:~$ sudo aircrack-ng capture-01.cap Opening capture-01.cap Read 1524 packets. # BSSID ESSID Encryption 1 AA:BB:CC:DD:EE:FF MyNetwork WPA (1) handshake

08 WPS Detection

WPS-enabled APs are targets for Pixie Dust and PIN brute force attacks (Module 11). Detecting WPS during recon is essential.

WPS detection with airodump-ng and wash
# airodump-ng: --wps flag adds WPS column kali@kali:~$ sudo airodump-ng --wps --manufacturer wlan0mon BSSID ... WPS MANUF ESSID AA:BB:CC:DD:EE:FF ... 2.0 TP-LINK HomeNet ← WPS 2.0 enabled 00:11:22:33:44:55 ... 1.0 Netgear OldAP ← WPS 1.0 (Pixie Dust target!) 11:22:33:44:55:66 ... -- ASUS Office ← WPS disabled # wash: dedicated WPS scanner (included with reaver) kali@kali:~$ sudo wash -i wlan0mon BSSID Ch dBm WPS Lck Vendor ESSID AA:BB:CC:DD:EE:FF 6 -45 2.0 No TP-LINK HomeNet 00:11:22:33:44:55 1 -52 1.0 No Netgear OldAP ← unlocked = attackable 11:22:33:44:55:66 11 -67 2.0 Yes ASUS Office ← locked = brute-force locked out # Lck=Yes means too many failed PINs triggered WPS lockout
⚡
Attack selection: WPS 1.0 + Lck=No → immediate Pixie Dust attempt. WPS 2.0 + Lck=No → PIN brute force. WPS Lck=Yes → wait for lockout to expire (usually 60 sec) or try a different time. Full attack in Module 11.

10 Extracting Recon Data with tshark

After a capture session, tshark lets you programmatically extract intelligence from the .cap file without opening Wireshark.

tshark intelligence extraction
# All unique SSIDs from beacon frames kali@kali:~$ tshark -r capture-01.cap -Y "wlan.fc.type_subtype==8" -T fields -e wlan_mgt.ssid | sort -u HomeNet OfficeWifi CoffeeShop # All client probe requests (PNL leakage) kali@kali:~$ tshark -r capture-01.cap -Y "wlan.fc.type_subtype==4 && wlan_mgt.ssid" -T fields -e wlan.sa -e wlan_mgt.ssid | sort -u 11:22:33:44:55:66 HomeWiFi 11:22:33:44:55:66 OfficeNet 2a:bb:cc:dd:ee:ff (randomized MAC probing) # Check EAPOL handshake frames (for cracking readiness) kali@kali:~$ tshark -r capture-01.cap -Y "eapol" -T fields -e frame.number -e eapol.keydes.msgnr -e wlan.sa -e wlan.da 42 1 AA:BB:CC:DD:EE:FF 11:22:33:44:55:66 (M1: AP→Client) 43 2 11:22:33:44:55:66 AA:BB:CC:DD:EE:FF (M2: Client→AP ← crackable!) 44 3 AA:BB:CC:DD:EE:FF 11:22:33:44:55:66 (M3: AP→Client) 45 4 11:22:33:44:55:66 AA:BB:CC:DD:EE:FF (M4: Client→AP) # All 4 messages captured = complete handshake
